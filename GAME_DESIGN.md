# ゲーム化方針メモ

## 今回の判断

この作品は、戦闘システムより先に「本文を読ませる」「選択で扱いが変わる」「ステータス表示が出る」ことが重要なので、最初はHTML/CSS/JavaScriptだけの静的ノベルゲームにするのが安全です。

## 作品のゲーム化に向いている要素

- 画面端に出るステータス表示
- 未契約者、仮契約、洗礼、登録などの制度
- HP/MP、スキル獲得、状態異常
- 「首輪にも盾にもなる」という選択の重さ
- 魔法検証パートのコメディ性

## MVPの流れ

1. 第一話〜第四話を一本道で読めるようにする
2. 章末に選択肢を置く
3. ステータス欄を常時表示する
4. BAD ENDや別ルートを小さく追加する
5. 背景・立ち絵・BGMを後から足す
