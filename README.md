# Novel Game Prototype

ブラウザで動く3D探索型ノベルRPGの試作です。現在は、佐藤雄二が森の街道で目覚め、荷車襲撃・商人救助・王都アウレリア入城・ギルド仮登録まで進める導線を遊べます。

## 現在できること

- タイトル画面から開始
- 王都へ続く森の街道から開始
- 森の街道、荷車襲撃現場、王都門を探索
- 黒毛の噛み犬による荷車襲撃イベント
- ファイアボールまたは囮行動で商人を助ける分岐
- 商人紹介状を得て王都ギルド登録へ進む導線
- 王都アウレリア全域マップを探索
- 城壁、北門、中央広場、王城、ギルド、教会、訓練場、市場、貴族街、兵舎、スラム風区画を配置
- 中世ヨーロッパ風の家屋、木組み、梁、看板、煙突、窓、屋台、木箱、樽、井戸、街灯を配置
- 怪しい路地裏エリア、暗いランタン、干し布、隠し扉、情報屋NPCと専用会話
- 改善した馬車が大通りを走る交通表現（車輪、車軸、馬、手綱、積み荷）
- 通行人、警備兵、商人、司祭、信徒、冒険者、鍛冶職人、情報屋が街区ごとに滞在・巡回する生活感表現
- 建物、城壁、木、岩、荷車、屋台、家具、歩行者、馬車などの当たり判定と押し戻し
- ギルド内部に棚・カウンター・机・木箱を追加
- 3Dファイアボールを任意発射
- 1人称視点で照準方向に火球を撃つアクション、MP消費、クールタイムUI、命中時の発光・爆発・カメラ揺れ・ヒットストップ
- 回避アクション
- デバッグ用の空中浮遊・高速移動モード
- 頭、髪、前髪、目、胴体、腕、手、脚、靴、マント、ベルトを整理したアニメ調低ポリ人型モデル
- WASDで移動、矢印キーで視点操作
- Shift / R2でスタミナ消費型ダッシュ
- 1人称 / 3人称カメラ切替
- DualShock 4 / 標準Gamepad API対応
- 会話選択肢をDualShock 4で上下選択して決定
- 入口・掲示板などに発光マーカーを配置
- 会話の選択肢でステータス・目的・クエスト進行を変化

開発確認用に `?map=plaza`、`?map=guildHall`、`?map=church`、`?map=trainingGround` を付けると各マップから直接開始できます。`?debug=1` を併用すると内部確認用APIも有効になります。

## 軽量化オプション

通常表示では、ブラウザの描画解像度を中程度に抑える `render=medium` 相当の上限を入れています。見た目を大きく落とさず、GitHub Pages上での王都マップの重さを下げるためです。

| URLパラメータ | 内容 |
|---|---|
| `?render=low` | 描画解像度をより低くして軽くする |
| `?render=medium` | 既定。見た目と軽さのバランス |
| `?render=high` | 高解像度寄り。重い場合あり |

例：`?map=plaza&render=low` で王都を軽量表示できます。

## 今回の設計方針

- 「広いだけ」の王都ではなく、通行人・馬車・市場・路地裏・生活小物で密度を上げる
- 建物や木をすり抜ける違和感を減らすため、基本的なAABB当たり判定を追加
- 中世ヨーロッパ風の城壁都市として、城壁・塔・市場・貴族街・スラム風区画・ギルド・教会を区別する
- ファイアボールの場面を王都内ではなく「王都へ続く森の街道」に置く
- 広さだけでなく、街道分岐、看板、門、発光マーカーで自然に誘導する
- 「遅さは罪」を意識し、広いマップ用にダッシュとデバッグ高速移動を用意
- なろう系の弱点である“ご都合主義の無双”を避けるため、商人紹介状・身分保証・ギルド審査を進行の根拠にする
- 戦闘操作は今後のために、DualShock 4の□/○/△/R2へアクションを割り振る方向に寄せる
- 重い場合でも王都の密度を単純に削らず、まず描画解像度上限で負荷を下げる

```text
├── story.js         # NPC会話・選択肢・クエスト・ステータス
├── THREE_PROTOTYPE.md
└── assets/
    ├── models/      # 将来の3Dモデル置き場
    ├── textures/    # 将来のテクスチャ置き場
    ├── bgm/         # BGM置き場
    └── se/          # 効果音置き場
```
