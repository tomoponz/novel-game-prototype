## パフォーマンス（重要・誤解防止）

- 本セッションのプレビューは **WebGLがSwiftShader（CPUソフトウェア描画）** で動作しており、密インスタンス都市(plaza)は **2fps前後** に落ちた。
- ただし **同一ブラウザで origin/main（本PR変更前）も plaza 2fps** であることを確認済み。→ **低FPSは本PRのUI変更が原因ではなく、ソフトウェア描画の制約**。
- 小規模マップ(guildHall)は同セッションで **120fps**。JSループ・描画ロジックは健全。
- 本PRのUI変更は毎フレーム負荷をほぼ増やさない（下部HUDのstamina%テキスト更新程度）。GPU有効の実機/GitHub Pagesでは従来通り。

## 既知の問題 / 次PR候補

- 酒場・記録所・武器屋などは「会話のみ」（内部マップ未実装）。
- ホットバーは見た目選択のみ（実アイテム使用処理は未実装）。
- HPは現状ダメージ処理がないため常に満タン表示。
- メニューのMap欄はアスキー簡易マップ（本格ミニマップは将来）。
- 街並みの大規模改善（区画の単調さ、施設誘導サイン強化）は本PRの範囲外として保留。
