# 王都アウレリア 内部施設拡張プラン (Capital Interior Expansion Plan)

関連Issue: #47(ギルド) / #48(宿屋・酒場) / #49(商会・市場) / #50(北門詰所) / #51(住宅・路地)。
正典の世界設定は `docs/world/kingdom-overworld-plan.md`（魔導王国＝世界、王都＝中央ハブ、
4ブロックは王都中心部のみ、魔法学院は王国内の一施設）。

## 0. 拡張方針

- 王都内の施設が「イベント用の小部屋」に見えると、制度機関としての説得力が弱い。
  そこで**中心部の汎用施設を中規模区画へ拡張**し、役割・記録・許可の運用が空間で伝わるようにする。
- **外観と内部サイズは厳密一致でなくてよい**。RPG的に内部マップは広くてよい（外観＝王都の一施設 / 内部＝役割ごとに拡張された専用マップ）。
- 一度に巨大化しない。1施設あたり **3〜6区画 / 2〜6 NPC / 1〜3 看板・掲示 / 既存進行と連動する導線**。
- 軽量維持: 大量NPC・物理エンジン・重アセットなし。既存の軽量プリミティブ／看板／簡易家具中心。`?render=low` 維持。
- 既存進行（guild_apply / mana_test / provisional / merchant / gate）と外付けJS
  （guild_anomaly / north_gate_check / progress_guard / next_destination / permit_ledger /
  record_update_toast / world_map）を壊さない。

## 1. ギルド会館（#47, map `guildHall`）

労働・冒険者管理機関に見せる。室を 16×15 → **28×22** に拡張。

| 区画 | 内容 |
|---|---|
| 受付カウンター | 既存導線（`guild_reception`）。表札「受付」 |
| 依頼掲示板 | 2枚の掲示板＋閲覧導線（`guild_board`）。F級単独不可・無登録禁止の但し書き |
| 待合スペース | ベンチ・卓・待機NPC（職員/冒険者/依頼人） |
| 魔力測定室 | 低い間仕切りで区切る。水晶は既存座標（`mana_measure`） |
| ギルドマスター室前 | 扉と表札（`guildmaster`） |
| 外門練習場 案内 | 「→ 外門練習場(王都南東)」看板 |

不変: 受付・水晶・ギルドマスターの座標と dialogue（進行・guild_anomaly と整合）。

## 2. 北門・衛兵詰所（#50, map `forestRoad` 門前）

一時通行が制度上の検問であることを空間で見せる（`gateCheckpoint`）。

| 区画 | 内容 |
|---|---|
| 検問カウンター | 門前の検問台 |
| 記録机 | 記帳台＋記録係NPC（`gate_record`）。紹介状番号を控える |
| 武器棚/装備置き場 | 槍立て・棚 |
| 待機ベンチ | 通行待ちの席 |
| 衛兵詰所 | GATE POST 小屋（看板「衛兵詰所」） |
| 事情聴取室 | 小部屋＋掲示（`gate_notice`） |
| 王都側出口 | 北門の遷移（`north_gate` → plaza） |

不変: `merchant`/`gate` 進行、紹介状必須、`progress_guard` / `north_gate_check.js`。

## 3. 宿屋・酒場「曲がった匙亭」（#48, map `inn`）

章間の滞在・情報拠点。室を 14×14 → **24×20** に拡張。

| 区画 | 内容 |
|---|---|
| 受付 | 女将マルタ（`inn_marta`）。表札 |
| 食堂/酒場 | 卓・客席・樽・酒棚 |
| 客席 | ベンチ群 |
| 客室入口 | 表札「客室入口」 |
| 噂掲示板 | 閲覧導線（`inn_board`） |
| 2階風階段 | 客室(2階)への階段＋看板 |

噂NPC: 旅商人（遺跡/南港）/ 宿の冒険者（査定）/ 学院生（禁書区）/ 教会関係者（地下記録室）/ 非番の兵士（王城呼び出し）。
不変: `inn_marta` 導線。各噂は進行フラグなしの flavor。

## 4. 住宅街・路地（#51, 新map `backstreet`）

王都を「施設が置いてあるマップ」ではなく、人が暮らす街に見せる小区画。

| 要素 | 内容 |
|---|---|
| 狭い路地 | 両側に迫る住宅＋細い路 |
| 井戸/水場 | 井戸（`backstreet_well`） |
| 生活物 | 洗濯物・木箱・樽・篝火 |
| 市民NPC | 路地の住人/井戸端の女/古株/小商人/声をひそめる男 |
| 裏口/行き止まり | 行き止まり・施錠された裏口 |
| 情報屋 | `alley` dialogue 維持（alleyクエスト進行不変） |

市民会話: 身元なしへの警戒 / ギルド登録者の噂 / 教会記録係の話 / 商会信用の話 / 王城監視の噂。
導線: 広場「裏路地に入る」→ backstreet、出口で大通りへ。world_map の「住宅街/路地」から travel 可能。

## 5. 商会・市場店舗（#49, 実装済み: feature/merchant-market-interior-expansion）

商会信用が、単なる数値ではなく**帳簿・保証・配送・取引・倉庫**で管理されて見えるよう、
新map `merchantOffice`（エドリック商会 王都支所）を追加。広場の市場通り・商会方面から入る。

| 区画 | 内容 |
|---|---|
| 商会受付 | 信用照会（`merchant_desk`）。紹介状＝一時的な信用 |
| 帳簿机 | 取引・配送・揉め事を記録（`merchant_ledger`）。ギルド・王都と照合 |
| 倉庫入口 | 在庫・配送票。**Merchant Trust 3以上でゲート**（`merchant_warehouse`） |
| 依頼掲示板 | 荷物配送/書類提出/倉庫整理/取引記録/露店トラブル（`merchant_board`） |
| 商会責任者室 | 情報・保証の取引。**Merchant Trust 5以上でゲート**（`merchant_backroom`） |
| 荷物置き場 | 木箱・樽 |
| 市場フロント | 雑貨屋（`shop_general`）・鑑定屋（`shop_appraise`） |

NPC: 商会受付・帳簿係・倉庫番・配送係・商会責任者・雑貨屋の主・鑑定屋。
入域ゲートは `progress_guard.js` の Merchant Trust ルールで実装（信用が低い間は倉庫・奥部屋に入れず理由を表示）。
small依頼は掲示板に提示するのみで本格クエスト化は未実装（将来導線）。

market の盗難イベント（`market`）・宿屋の噂と接続し、南港市場（#52 ロードマップ第4章）への布石とする。
