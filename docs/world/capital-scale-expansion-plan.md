# 王都アウレリア中心部 スケール拡張計画

関連Issue: #17 王都アウレリア中心部を現在の約4倍規模へ段階拡張する。

## 1. 4ブロック構想の定義

ここでいう「4ブロック構想」は、王都アウレリア中心部を現在の約4倍規模へ段階的に広げるための制作単位である。

これは思想別・勢力別に街を4分割する設計ではない。

避ける表現:

- 王権区 / 学院区 / 教会区 / 商業区として完全分離する
- 色分けされたテーマ地区として扱う
- 4つの思想別国家や別勢力圏のように扱う
- 1つの巨大マップを単純に4倍化して重くする

採用する表現:

- 中央広場から街路が自然に伸びている
- 行政施設はある程度まとまるが、周囲に宿屋・商店・住宅・路地が混在する
- 市場の近くにも住宅と井戸がある
- 学院方面にも学生向け店舗や一般市民の生活がある
- 教会方面にも記録所・治療所・住宅・路地が混ざる
- 王都内の移動は、方面・通り・近くに多い施設で案内する

## 2. Phase 1: 中央広場周辺の導線追加

今回のPR範囲。plaza を巨大化せず、中央広場周辺だけに次を追加する。

- 行政庁・王城区方面、市場通り・商会方面、学院通り、大聖堂・記録所方面、宿屋・裏路地方面の案内
- 広場外周に短い街路の受けを追加
- 小店舗/屋台、住宅風建物、井戸、掲示板、木箱、樽を少量追加
- 進行フラグを変えない市民flavor会話を追加

目的は「王都がこの先へ広がる」ことを示すことであり、4地区を完成させることではない。

## 3. Phase 2: 行政通り・許可局・記録官窓口

今回のPR範囲。中央広場から王城区・行政庁方面へ向かう軽量サブマップ `administrativeStreet` を追加する。

- `plaza` の「行政庁・王城区方面」導線から入る
- 許可局の窓口、記録官窓口、警備詰所、王城区方面へ続く門を置く
- 王城区本体はまだ実装しない
- 港湾、辺境砦、王城区など用途別許可の概念を示す
- 住宅、小商店、屋台、井戸、待機列、市民NPCを混在させる

行政施設がある程度まとまることは許容する。ただし「王権区」のような分離地区にはせず、役所の近くにも普通の生活と小商いがある通りとして扱う。

## 4. Phase 3: 近接施設の厚みを増やす

既存の `guildHall` / `inn` / `merchantOffice` / `backstreet` を、plaza から自然に出入りできる近接施設として強化する。

- `guildHall`: 受付、掲示板、待合、測定室、ギルドマスター室前を拡張
- `inn`: 食堂、客席、噂掲示板、客室入口を拡張
- `merchantOffice`: 帳簿机、倉庫、鑑定/雑貨、小口依頼を拡張
- `backstreet`: 井戸、洗濯物、裏口、情報屋、生活路地を拡張

これらは独立したテーマ地区ではなく、王都中心部の近接サブマップとして扱う。

## 5. Phase 4: 大型施設の外構を厚くする

既存の `academyCampus` / `churchGrounds` を、王都内にある大規模施設の入口として育てる。

- `academyCampus`: 校庭、講義棟、塔、庭、門、学生向け店舗へ段階拡張
- `churchGrounds`: 大聖堂前庭、記録所、庭園、治療所、参拝者の導線へ段階拡張

どちらも「学院だけの区」「教会だけの区」にはしない。周辺には住民、商店、宿、記録手続き、路地を混在させる。

## 6. Phase 5: 中心部の約4倍規模化

Phase 1〜4で導線とサブマップが安定したあと、王都中心部を複数の軽量サブマップに分けて拡張する。

- 中央広場をハブとして維持
- 王城方面、北門方面、市場方面、学院/教会方面、住宅/路地方面を自然につなぐ
- world_map / next_destination / progress_guard / permit_ledger と接続する
- 1枚の巨大plazaに全部を詰め込まず、GitHub Pagesで動く軽さを維持する

## 7. 既存マップとの接続

| 既存マップ | 役割 | 現在の扱い |
|---|---|---|
| `plaza` | 中央広場と城下町のハブ | 広場周辺の案内・街路・生活感を追加 |
| `administrativeStreet` | 行政通り・許可局・記録官窓口 | Phase 2で追加。王城区方面導線を示すが王城区本体は未実装 |
| `guildHall` | 冒険者登録・依頼・信用 | 既存入口を維持 |
| `inn` | 宿泊・噂・休憩 | 宿屋方面の案内で導線を補強 |
| `merchantOffice` | 商会・帳簿・倉庫 | 市場通り・商会方面として案内 |
| `backstreet` | 住宅・裏路地・非公式情報 | 宿屋・裏路地方面として案内 |
| `academyCampus` | 学院外構・講義棟入口 | 学院通りとして案内 |
| `churchGrounds` | 大聖堂外構・記録所入口 | 大聖堂・記録所方面として案内 |

## 8. 実装上の制約

- `render=low` を維持する
- 大量NPCや大量小物を置かない
- 物理エンジンや新フレームワークを導入しない
- 進行フラグ、許可証UI、ミニマップ、world_map、next_destination を壊さない
- Phase 1では王都全体の巨大リメイクをしない
