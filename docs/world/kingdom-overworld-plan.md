# 魔導王国 全体マップ構想 (Kingdom Overworld Plan)

関連Issue: #52(全体マップ構想), #53(ワールドマップUI), #54(入域条件), #55(長期ロードマップ), #17(王都4ブロック)

## 0. 最重要の前提（フレーミング訂正）

過去の表現で「王立魔法学院」が世界の中心のように見える箇所がありましたが、それは誤りです。
本ドキュメントを**正典(canonical)**とし、以下を世界設定の基準とします。

- 舞台は、魔法が高度に発達した **「魔導王国」** 全体である。
- **魔法学院は、魔導王国内にある教育・研究施設の一つ**にすぎない。世界・国家の中心ではない。
- **4ブロック構想は、王国全体の地理区分ではなく、王都アウレリア中心部の都市構造**である（#17）。
- 王都の外には、港町・教会領・辺境砦・農村・森・沼地・遺跡・外縁危険地帯などが存在する。

```
魔導王国全体
└ 王都アウレリア（中央ハブ）
   └ 王都中心部の4ブロック構想（#17 / 都市スケール）
      ├ 制作単位A: 中央広場と混在街路
      ├ 制作単位B: 行政通り・許可局・記録官窓口
      ├ 制作単位C: 近接施設と市場/住宅/路地の厚み
      └ 制作単位D: 学院/大聖堂方面の外構と周辺生活路
```

上記は制作順とスケール管理の単位であり、思想別・勢力別・色分けされた4地区ではない。
行政施設や学院・教会系施設が近くにまとまることはあるが、周囲には住宅、商店、宿、路地、市民生活を混在させる。

魔法学院は「王都または王都周辺にある重要施設」であり、**世界全体を代表する地域名にはしない**。

## 1. 世界の3層構造

| 層 | 単位 | UI | 例 |
|---|---|---|---|
| 1. 魔導王国全体 | ワールドマップ | `world_map.js`（本PRで追加） | 王国図、地域選択 |
| 2. 主要地域 | 地域マップ | 既存3Dマップ + 将来追加 | 王都中心区, 森の街道, 学院キャンパス, 大聖堂前庭 |
| 3. 建物/施設 | 専用内部マップ | 既存3D内部マップ | ギルド本部, 学院講義棟, 教会記録所, 宿屋 |

王都アウレリアは**中央ハブ**。王都外地域は、章進行・許可・信用・記録によって段階的に解放する。

## 2. 中心テーマ

単なるオープンワールドではない。中心テーマは:

- 身分 / 信用 / 記録 / 許可 / 所属機関 / 魔導王国による管理
- **記録されるほど便利になるが、逃げにくくなる**構造

地域解放はレベル制ではなく、次の組み合わせで決める（詳細は `region-access-rules.md`）:

- ギルド登録状態 / 商会信用(Merchant Trust) / 教会照会状態 / 学院観察状態
- 王城区からの行動許可(Crown Record) / 地域固有の危険度 / 章進行フラグ

## 3. 地域構成

凡例:
- **位置**: ワールドマップ上の方角。
- **管理**: その地域を統治・管理する勢力。
- **入域条件**: そこへ入るために満たすべき条件（詳細は region-access-rules.md）。
- **章**: 主に関わる章（long-term-roadmap.md）。
- **優先度**: 実装優先度（H=高/M=中/L=低、★=実装済みの既存3Dマップ）。

## 4. 実装方針（重要）

- **いきなり全地域を3D実装しない。** まずは docs（本書 / region-access-rules / long-term-roadmap）と軽量ワールドマップUI(`world_map.js`)から。
- 既存3Dマップ（plaza / forestRoad / trainingGround / academyCampus / academy / churchGrounds / church / guildHall / inn）は地域層(層2/層3)としてそのまま流用。
- ワールドマップUIは**DOM/CSS中心・軽量**。大きな画像や重い描画は使わない。`?render=low` を維持。
- 将来は手描き風王国マップ（参考画像のような）へ段階的に置換可能な設計とする。

## 5. 既存システムとの非競合

- `permit_ledger.js`(Pキー) … 個人の許可・信用の「縦の記録」。ワールドマップは「横の地理」。役割分担。
- `record_update_toast.js` … クエスト完了通知。地域解放時にも将来連動可能（follow-up）。
- `next_destination.js` / minimap … 現在マップ内の現在地・次目的地。ワールドマップは王国規模の俯瞰。
- `progress_guard.js` … E操作の入域ゲート。ワールドマップの解放判定は同じ条件をミラーする（二重管理しない）。

## 6. 将来拡張（follow-up）
